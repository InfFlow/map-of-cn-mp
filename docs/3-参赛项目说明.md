# Map of Us · 我们的地图

> 一个有温度、会讲故事的情侣旅行记录小程序。
> 不只是记录去过哪里，更把走过的路，变成值得反复翻看的回忆。

---

## 一、项目定位

市面上的旅行/打卡类产品大多停留在「工具」层面——记录坐标、堆叠照片。
**Map of Us 的差异化在于「情感化 + AI 叙事」**：把冰冷的足迹数据，通过 AI 和精心设计的交互，转化为有温度的共同回忆。

目标用户：**情侣 / 伴侣**，记录两个人一起走过的城市与时光。

设计语言：**黑白杂志极简风（Kinfolk 风格）**——纸白底、墨黑字、衬线大标题，克制而高级，全程无彩色干扰。

---

## 二、核心亮点（评审记忆点）

### 1. AI 全场景叙事
不是「接个聊天框」式的伪 AI，而是把 AI 深度嵌入每个场景：
- **旅行故事生成**：聚合全部足迹/手记/照片，写成一篇完整的旅行回忆散文，可导出长图
- **照片手记**：看照片自动生成第一人称旅行手记 + 情绪/场景标签
- **精华筛选**：从海量照片里 AI 评选「最佳时刻 / 最爱地点」
- **每日灵感**：每天一句基于真实足迹生成的旅行感悟
- **心愿攻略 / 下一站推荐**：基于已去过的地方，AI 推荐目的地和玩法
- **预算分析**：AI 解读消费结构给出建议

### 2. 情感化设计
- **时间胶囊**：写给未来的信，到期才能开启
- **今天 N 年前**：打开 App 自动唤起「N 年前的今天，我们在某地」
- **纪念日**：在一起第 N 天实时计算、周年倒计时
- **两人足迹对比**：「一起去过 / 你去过 / TA 去过」

### 3. 多维数据可视化（纯 Canvas / 原生）
- 中国地图省份高亮 + 城市标记 + 旅行轨迹连线
- 足迹热力图、34 省打卡进度、旅行年历、标签云
- 数据概览大屏（6 维数字墙 + 年度趋势 + 季节分布）
- 花费饼图、回顾海报（三种风格，可分享）

### 4. 工程完整度
- **AI 能力后台一键开关**：审核期间关闭呈现纯净版，通过后开启完整体验
- **密钥安全**：AI / 地图密钥全部在服务器 `_private` 目录，前端零暴露
- **离线缓存**：弱网/断网自动读本地缓存
- **统一设计系统**：CSS 变量色板 + 4 档圆角规范 + 全局动效类

## 四、技术架构

```
微信小程序（原生）
   │  wx.request / 自定义 tabBar / Canvas 2D
   ▼
PHP 后端（ql.hlat.xyz）
   ├── admin_api.php   统一业务接口（足迹/纪念日/心愿/胶囊/打卡/AI…）
   ├── journeys.php    足迹数据
   ├── 网页运营后台     /admin（足迹/纪念日/菜品/系统配置）
   └── _private/       密钥与配置（HTTP 不可访问）
   ▼
MySQL  +  DeepSeek API（AI）  +  高德 API（地理/天气/路线）
```

**前端**：微信原生，无第三方框架。统一 CSS 变量设计系统，全局动效（数字滚动 / 逐项浮现 / iOS 级压感 / 杂志线条空状态插画）。

**后端**：PHP + MySQL，PDO 参数化查询防注入；AI / 地图密钥服务端持有；图片上传校验 MIME + 大小。

**AI**：DeepSeek（文本生成与结构化 JSON 输出），覆盖手记、故事、攻略、灵感、分析、推荐六类场景。

## 五、设计系统

- **色板**：纸白 #f4f1ea · 墨黑 #1b1712 · 中性灰阶，单色无彩
- **字体**：衬线（Georgia/宋体）做大标题与数字，无衬线做正文
- **圆角**：sm 6 / md 12 / lg 18 / full 999 四档规范
- **动效**：入场渐显上滑、列表逐项浮现、数字缓动滚动、iOS 回弹压感
- **细节**：报头双线、栏目中英标题、图注编号（PLATE/COVER）、首字下沉、骨架屏

---

## 六、安全与合规

- AI 功能后台开关，可在审核期关闭，避免「深度合成 / AI 问答」类目风险
- 所有密钥服务端持有，前端不暴露
- 数据库参数化查询，上传校验，CSRF 保护（网页后台）
